High-level container for resources managed by the RenderBackend.

This includes a variety of things, including images, fonts, and glyphs, which may be stored as memory buffers, GPU textures, or handles to resources managed by the OS or other parts of WebRender.

The rasterizer associated with the current scene.

blob_image_rasterizer_produced_epoch: BlobImageRasterizerEpoch

An epoch of the stored blob image rasterizer, used to skip the ones coming from low-priority scene builds if the current one is newer. This is to be removed when we get rid of the whole "missed" blob images concept. The produced one gets bumped whenever we produce a rasteriezer, which then travels through the scene building and eventually gets consumed back by us, bumping the consumed epoch.

blob_image_rasterizer_consumed_epoch: BlobImageRasterizerEpochdeleted_blob_keys: VecDeque<Vec<BlobImageKey>>

A log of the last three frames worth of deleted image keys kept for debugging purposes.

dirty_image_keys: HashSet<ImageKey, BuildHasherDefault<FxHasher>>

A set of the image keys that have been requested, and require updates to the texture cache. Images in this category trigger invalidations for picture caching tiles.

active_image_keys: HashSet<ImageKey, BuildHasherDefault<FxHasher>>

A set of the image keys that are used for render.
